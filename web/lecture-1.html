Course Overview & The Shell

动机 (Motivation)

正如原文所说，计算机在辅助重复性任务方面表现出色。然而，我们常常忽略了这一点同样适用于我们使用计算机的方式，而不仅仅是我们希望程序执行的计算。我们手头有大量的工具可以提高工作效率，解决更复杂的问题。但许多人只使用了这些工具的一小部分，仅仅通过死记硬背一些"魔法咒语"来应付工作，遇到问题时则盲目地从互联网上复制粘贴命令。

这门课正是为了解决这个问题。

我们希望教你如何最大限度地利用你已知的工具，向你展示可以添加到工具箱中的新工具，并希望能够激发你探索（甚至构建）更多工具的热情。我们相信这是大多数计算机科学课程中"缺失的一学期"。

拓展思考：

  • 自动化思维： 学习 Shell 的核心之一是培养"自动化思维"。当你发现自己重复执行某个命令序列超过两三次时，就应该考虑如何将其自动化（例如，编写一个简单的脚本）。
  • DevOps 与 SRE 文化： 在现代软件开发和运维（DevOps, Site Reliability Engineering）中，Shell 脚本和命令行工具是实现自动化部署、监控、管理和故障排除的基础。

由于时间有限，我们无法像完整课程那样详细地介绍所有工具。在可能的情况下，我们会为你指出深入研究某个工具或主题的资源，但如果有什么特别吸引你的东西，请不要犹豫，向我们寻求指点！

主题1：Shell (Topic 1: The Shell)

Shell 是什么？(What is the shell?)

如今的计算机有多种命令界面：精美的图形用户界面（GUI）、语音界面，甚至AR/VR无处不在。这些界面对于80%的用例来说非常棒，但它们在功能上往往有根本性的限制——你无法按下不存在的按钮，或发出未被编程的语音命令。为了充分利用计算机提供的工具，我们必须回归传统，使用文本界面：Shell。

几乎所有你能接触到的平台都以某种形式提供了 Shell，许多平台甚至有多种 Shell 供你选择。虽然它们在细节上可能有所不同，但其核心大致相同：它们允许你运行程序，向程序提供输入，并以半结构化的方式检查其输出。

在本讲座中，我们将重点介绍 Bourne Again SHell，简称 "bash"。这是使用最广泛的 Shell 之一，其语法与你在许多其他 Shell 中看到的相似。要打开一个 Shell 提示符（你可以在其中输入命令），你首先需要一个终端 (Terminal)。你的设备可能已经安装了一个终端，或者你可以很容易地安装一个。

拓展技巧与现代方法：

为什么选择 Shell？
  • 自动化与脚本： 实现复杂任务的自动化。
  • 远程访问： SSH 交互的主要方式。
  • 资源效率： 通常消耗更少系统资源。
  • 精确控制： 提供更细致的系统控制。
  • 可组合性： 通过管道连接小程序。
常用的 Shell：
  • Bash: Linux/macOS 默认，兼容性好。
  • Zsh: 功能更强，自动补全、主题 (Oh My Zsh)。
  • Fish: 用户友好，智能建议，语法高亮。
  • PowerShell: Windows 现代化 Shell。
Windows 用户的选择：
  • WSL (WSL2): 强烈推荐，运行完整 Linux。
  • Git Bash: Git for Windows 附带。
  • Cygwin/MinGW: 更全面的 Unix 工具集。
  • Windows Terminal: 现代化终端应用。

使用 Shell (Using the shell)

当你启动终端时，你会看到一个通常如下所示的提示符：

missing:~$

这是 Shell 的主要文本界面。它告诉你当前在名为 `missing` 的机器上，并且你的"当前工作目录"（你当前所在的位置）是 `~`（"home"目录的简写）。`$` 符号告诉你你不是 root 用户（稍后会详细介绍）。在这个提示符下，你可以输入命令，然后 Shell 会解释并执行它。最基本的命令是执行一个程序：

missing:~$ date
Fri 10 Jan 2020 11:49:31 AM EST
missing:~$

这里，我们执行了 `date` 程序，它（也许不足为奇地）打印出当前的日期和时间。然后 Shell 会提示我们输入下一个要执行的命令。我们也可以带参数执行命令：

missing:~$ echo hello
hello

在这个例子中，我们告诉 Shell 用参数 `hello` 执行 `echo` 程序。`echo` 程序只是简单地打印出它的参数。Shell 通过按空格分割命令来解析它，然后运行第一个单词指示的程序，并将后续的每个单词作为程序可以访问的参数提供。如果你想提供包含空格或其他特殊字符的参数（例如，名为 "My Photos" 的目录），你可以用 `'` 或 `"` 来引用参数 (`"My Photos"`)，或者只用 `\` 来转义相关字符 (`My\ Photos`)。

拓展技巧与现代方法 (提示符、引号、PATH、type):

理解提示符 (Prompt)
  • 通常格式为 `username@hostname:current_directory$`
  • `~` 代表用户主目录
  • `$` 普通用户, `#` 超级用户
  • 高度可定制 (PS1, Starship, Powerlevel10k)
引号的使用
  • 单引号 (`'`): 强引用，字面处理
  • 双引号 (`"`): 弱引用，变量/命令替换会被解释
  • 反斜杠 (`\`): 转义字符
$PATH 环境变量

Shell 查询 `$PATH` 环境变量来寻找程序。

echo $PATH
which echo
  • 临时添加: `export PATH="/new/path:$PATH"`
  • 永久修改: 编辑 `~/.bashrc` 或 `~/.zshrc`
`type` 命令

比 `which` 更全面，显示命令类型 (别名, 内建, 函数, 文件)。

type cd
type ls

在 Shell 中导航 (Navigating in the shell)

Shell 中的路径是一个分隔的目录列表（Linux/macOS用`/`，Windows用`\`）。以`/`（或Windows盘符如`C:\`）开头的为绝对路径，否则为相对路径。`.`代表当前目录，`..`代表父目录。

pwd              # 显示当前工作目录
cd /path/to/go # 切换目录
cd ..            # 切换到上级目录
cd               # 切换到主目录 (~)

要查看目录内容，使用 `ls` 命令。其行为可以通过标志（以`-`开头）修改。

ls               # 列出当前目录内容
ls /home         # 列出 /home 目录内容
ls -l            # 使用长列表格式显示
ls --help        # 查看 ls 命令的帮助

# ls -l 输出示例
drwxr-xr-x 1 missing  users  4096 Jun 15  2019 missing
# [权限] [链接数] [所有者] [所属组] [大小] [最后修改日期] [名称]

权限部分（例如 `drwxr-xr-x`）表示文件类型（`d`为目录，`-`为文件）以及所有者、所属组和其他人的读(`r`)、写(`w`)、执行(`x`)权限。

其他常用命令包括 `mv` (移动/重命名), `cp` (复制), `mkdir` (创建目录)。使用 `man <命令>` 查看手册页获取详细信息。

拓展技巧与现代方法 (导航、ls选项、man/tldr):

常用导航命令
  • `cd -`: 切换到上一个目录
  • `cd`: 返回主目录
  • `pushd/popd/dirs`: 目录栈操作
`ls` 常用选项
  • `-a`: 显示所有 (包括隐藏)
  • `-l`: 长列表格式
  • `-h`: 人类可读大小 (需配合 -l)
  • `-t`: 按时间排序
  • `-r`: 反向排序
  • `-R`: 递归列出
  • 常用组合: `ls -lathr`
获取帮助
  • `man <命令>`: 查看完整手册页
  • `tldr <命令>`: 查看简洁常用示例 (需安装)
  • `<命令> --help` 或 `-h`: 命令内置帮助
  • `apropos <关键词>`: 搜索相关手册页

文件和目录操作

除了导航，我们还需要管理文件和目录。常用的命令包括：

  • `mv <源> <目标>`: 移动或重命名文件/目录。
  • `cp <源> <目标>`: 复制文件/目录。
  • `mkdir <目录名>`: 创建新目录。
# 示例
mv old_name.txt new_name.txt  # 重命名
mv source_file.txt destination_folder/ # 移动
cp source_file.txt copy_file.txt # 复制文件
cp -r source_folder/ destination_folder/ # 复制目录 (需要-r选项)
mkdir new_directory # 创建目录

拓展技巧与现代方法 (文件操作):

`rm` 命令 (删除)
  • `rm <文件>`: 删除文件 (谨慎!)
  • `rm -r <目录>`: 递归删除目录 (非常谨慎!)
  • `rm -i`: 删除前交互式提示
  • `rm -f`: 强制删除，无提示 (极其谨慎!)
`cp` 和 `mkdir` 选项
  • `cp -r`: 递归复制目录
  • `mkdir -p`: 递归创建目录 (父目录不存在时一并创建)
获取帮助
  • `man <命令>`: 完整手册
  • `tldr <命令>`: 简洁示例
  • `<命令> --help/-h`: 内置帮助

连接程序 (Connecting programs)

在 Shell 中，程序有两个主要的"流"与之关联：它们的输入流 (input stream) 和它们的输出流 (output stream)。当程序尝试读取输入时，它从输入流中读取；当它打印某些内容时，它打印到其输出流。通常，程序的输入和输出都是你的终端，即你的键盘作为输入，屏幕作为输出。但是，我们也可以重定向这些流！

最简单的重定向形式是 `< file` 和 `> file`。它们分别允许你将程序的输入流和输出流重定向到文件：

missing:~$ echo hello > hello.txt
missing:~$ cat hello.txt
hello
missing:~$ cat < hello.txt
hello
missing:~$ cat < hello.txt > hello2.txt
missing:~$ cat hello2.txt
hello

如上例所示，`cat` 是一个连接文件的程序。当给定文件名作为参数时，它会按顺序将每个文件的内容打印到其输出流。但当 `cat` 没有给定任何参数时，它会将其输入流的内容打印到其输出流。

你也可以使用 `>>` 来追加到文件。这种输入/输出重定向真正发挥威力的地方在于管道 (pipes) 的使用。`|` 操作符允许你"链接"程序，使得一个程序的输出成为另一个程序的输入：

missing:~$ ls -l / | tail -n1
drwxr-xr-x 1 root  root  4096 Jun 20  2019 var
missing:~$ curl --head --silent google.com | grep --ignore-case content-length | cut --delimiter=' ' -f2
219

我们将在关于数据整理的讲座中更详细地讨论如何利用管道。

拓展技巧与现代方法 (流、重定向、管道):

标准流
  • `stdin` (0): 标准输入 (默认键盘)
  • `stdout` (1): 标准输出 (默认屏幕)
  • `stderr` (2): 标准错误 (默认屏幕)
重定向
  • `>`: 输出到文件 (覆盖)
  • `>>`: 输出到文件 (追加)
  • `<`: 从文件输入
  • `2>`: 错误输出到文件
  • `&>`: 输出与错误到文件
  • `/dev/null`: 丢弃输出。
    • `cmd > /dev/null` (忽略stdout)
    • `cmd 2> /dev/null` (忽略stderr)
    • `cmd &> /dev/null` (忽略所有)
Here Strings/Documents
  • `<<< "字符串"`: 将字符串作为输入
  • `<
管道 (`|`)

Unix 哲学核心：连接小程序处理文本流。

# 示例：查找最大的5个文件/目录
du -sh * | sort -rh | head -n 5

一个多功能且强大的工具

在大多数类 Unix 系统上，"root"用户拥有最高权限。通常我们不直接以 root 登录，而是使用 `sudo` 命令临时提升权限执行特定操作。使用 `sudo` 时务必谨慎！

Linux 系统通过 `/sys` 文件系统暴露内核参数。例如，可以通过读写 `/sys/class/backlight/.../brightness` 文件来控制屏幕亮度。macOS 和 Windows 没有 `/sys`。

# 错误示例：sudo 对重定向无效
# sudo echo 3 > /sys/class/backlight/intel_backlight/brightness 

# 正确示例：使用 tee 和管道
echo 3 | sudo tee /sys/class/backlight/intel_backlight/brightness

`tee` 命令从标准输入读取并写入到标准输出和文件，当与 `sudo` 配合时，`tee` 以 root 权限运行，从而可以写入受保护的文件。

拓展技巧与现代方法 (权限与系统交互):

`sudo` 的正确使用
  • `sudo <命令>`: 以 root 执行命令
  • `sudo -i` 或 `sudo su -`: 切换到 root 的交互式 shell (少用)
  • `sudoedit <文件>` 或 `sudo -e <文件>`: 安全编辑特权文件
`tee` 命令
  • 从标准输入读，写到标准输出和一个或多个文件。
  • `command | tee output.txt`
  • `command | sudo tee /privileged/file`
macOS 替代方案
  • 没有 `/sys`。
  • 屏幕亮度: 第三方工具 (`brightness`) 或 AppleScript。
  • 电池状态: `pmset -g batt`

后续步骤 (Next steps)

至此，你已经掌握了在 Shell 中完成基本任务的方法。你应该能够在文件系统中导航以查找感兴趣的文件，并使用大多数程序的基本功能。在下一讲中，我们将讨论如何使用 Shell 和许多方便的命令行程序来执行和自动化更复杂的任务。

拓展建议：

  • 练习，练习，再练习： 熟悉 Shell 的唯一方法就是多用。尝试用命令行完成你日常用 GUI 完成的任务。
  • 定制你的环境： 花时间配置你的 Shell (e.g., `.bashrc`, `.zshrc`)，设置别名 (aliases)，安装有用的工具。这将大大提高你的效率和使用体验。
  • 阅读手册： 遇到不熟悉的命令，记得 `man <命令>` 或 `tldr <命令>`。

练习 (Exercises)

本课程中的所有课程都附带一系列练习。有些练习要求你完成特定任务，而另一些则是开放性的，例如"尝试使用 X 和 Y 程序"。我们强烈建议你尝试一下。

对于本课程，你需要使用像 Bash 或 ZSH 这样的 Unix Shell。如果你在 Linux 或 macOS 上，则无需执行任何特殊操作。如果你在 Windows 上，你需要确保你运行的不是 `cmd.exe` 或 PowerShell；你可以使用 Windows Subsystem for Linux 或 Linux 虚拟机来使用 Unix 风格的命令行工具。要确保你正在运行适当的 Shell，可以尝试命令 `echo $SHELL`。如果它显示类似 `/bin/bash` 或 `/usr/bin/zsh` 的内容，则表示你正在运行正确的程序。

  1. 在 `/tmp` 目录下创建一个名为 `missing` 的新目录。

    mkdir /tmp/missing
# 验证: ls /tmp

  2. 查阅 `touch` 程序。`man` 程序是你的好朋友。

    man touch
# tldr touch (更简洁)

  3. 使用 `touch` 在 `missing` 目录中创建一个名为 `semester` 的新文件。

    touch /tmp/missing/semester
# 验证: ls /tmp/missing

  4. 将以下内容逐行写入该文件：

    #!/bin/sh
curl --head --silent https://missing.csail.mit.edu

    提示: 注意 `!` 的转义，可使用单引号或 Here Document。

    显示写入方法示例

    方法一: echo 和重定向

    echo '#!/bin/sh' > /tmp/missing/semester
echo 'curl --head --silent https://missing.csail.mit.edu' >> /tmp/missing/semester

    方法二: cat 和 Here Document

    cat < /tmp/missing/semester
#!/bin/sh
curl --head --silent https://missing.csail.mit.edu
EOF

  5. 尝试执行该文件 (`./semester`)。理解为何失败，并用 `ls -l` 查看权限。

    cd /tmp/missing
./semester # 可能会提示 Permission denied
ls -l semester

  6. 用 `sh semester` 执行脚本。解释为何这样可行。

    sh semester

    原因: `sh` 直接解释文件内容，无需文件有执行权限。

  7. 查阅 `chmod` 命令。

    man chmod

  8. 使用 `chmod +x semester` 使脚本可直接执行。解释 Shell 如何通过 shebang (`#!`) 知道用哪个解释器。

    chmod +x semester
./semester

  9. 使用管道和重定向，将 `semester` 脚本输出的 "Last-Modified" 日期写入 `~/last-modified.txt`。

    ./semester | grep -i "Last-Modified" > ~/last-modified.txt

  10. (Linux 用户) 读取电池电量或 CPU 温度。 (macOS 用户可尝试 `pmset -g batt`)

    # Linux 电池示例 (路径可能不同):
cat /sys/class/power_supply/BAT0/capacity

# Linux CPU 温度示例 (路径可能不同):
awk '{print $1 / 1000 "°C"}' /sys/class/thermal/thermal_zone0/temp

# macOS 电池示例:
pmset -g batt | grep -Eo "\d+%" | head -n 1
