键盘重映射 (Keyboard Remapping)

键盘是你与计算机交互的主要工具之一。与计算机的大多数方面一样，它是高度可配置的，进行一些个性化设置可以显著提高效率和舒适度。

基本重映射

最常见的定制是重新映射按键。例如，很多人发现 Caps Lock 键位置优越但功能鸡肋，因此将其重映射为更有用的键：

  • Caps Lock -> Ctrl (对 Emacs 或大量使用 Ctrl 快捷键的用户非常友好)
  • Caps Lock -> Esc (对 Vim 用户非常方便)
  • 交换 Ctrl 和 Meta (Win/Cmd) 键，以便在不同操作系统间保持肌肉记忆。

映射到自定义命令/宏

更进一步，你可以将按键或组合键映射到任意自定义命令或宏：

  • 绑定快捷键打开新的终端窗口或特定应用程序。
  • 设置宏快速输入常用文本（如邮箱地址）。
  • 绑定按键使电脑或显示器进入睡眠状态。

高级定制与工具

一些工具支持更复杂的定制，如根据按键时长执行不同操作（Tap vs Hold）、按键序列、或仅在特定应用中生效的映射。

常用工具:

macOS
  • Karabiner-Elements
  • skhd (常与 yabai 配合)
  • BetterTouchTool
  • 系统设置
Linux
  • xmodmap, setxkbmap, xkb
  • Autokey
  • kmonad (跨平台)
  • 桌面环境设置 (GNOME Tweaks, KDE)
Windows
  • PowerToys Keyboard Manager
  • AutoHotkey
  • SharpKeys
跨平台/固件级
  • QMK Firmware / VIA / Vial (用于支持的机械键盘)

守护进程 (Daemons)

守护进程 (daemon) 是指那些在后台持续运行、通常在系统启动时就开始执行的程序。它们不像普通程序那样需要用户交互启动，而是默默地提供各种系统服务。常见的例子有 sshd (SSH 服务)，nginx (Web 服务器)，数据库服务等。

服务管理

Linux (systemd)

  • 管理器: systemd
  • 控制命令: systemctl
    • status, start, stop, restart
    • enable (开机自启), disable
  • 配置文件: .service 文件 (位于 /etc/systemd/system/ 或 ~/.config/systemd/user/)

macOS (launchd)

  • 管理器: launchd
  • 控制命令: launchctl
  • 配置文件: .plist 文件 (位于 /Library/LaunchDaemons, /Library/LaunchAgents, ~/Library/LaunchAgents)

Windows (Services)

  • 管理工具: services.msc (图形化)
  • 命令行: PowerShell (Get-Service, Start-Service 等)

创建自定义服务文件（如 systemd 的 .service 或 launchd 的 .plist）可以让你将自己的应用程序或脚本作为后台服务来管理。

定时任务 (Scheduled Tasks)

定时任务允许你安排命令或脚本在特定时间或按固定间隔自动执行。

  • cron (Linux/macOS): 经典工具，使用 crontab -e 编辑任务列表，通过 crontab 表达式指定时间。
  • systemd Timers (Linux): 更现代、灵活的替代方案，通过 .timer 文件配置。
  • launchd Timers (macOS): 在 .plist 文件中使用 StartInterval 或 StartCalendarInterval。
  • 任务计划程序 (Task Scheduler) (Windows): Windows 内置的图形化定时任务工具。

FUSE (用户空间文件系统)

通常，文件系统的实现位于操作系统内核中。但 FUSE (Filesystem in Userspace) 是一种允许用户程序实现文件系统接口的机制。它充当内核和用户空间代码之间的桥梁，使得在用户态响应和处理文件系统操作成为可能。

这意味着你可以编写程序，将各种数据源或服务"挂载"为看起来像普通文件和目录的文件系统，然后使用标准的文件操作命令（`ls`, `cd`, `cat`, `cp` 等）或图形化文件管理器来与之交互。

有趣的 FUSE 应用示例

  • sshfs: 通过 SSH 将远程目录挂载到本地。
  • rclone mount: 将多种云存储服务 (Dropbox, Google Drive, S3 等) 挂载为本地文件系统。
  • gocryptfs / cryfs / encfs: 创建加密的覆盖文件系统。
  • kbfs (Keybase Filesystem): 端到端加密的分布式文件系统。
  • borgbackup mount: 挂载 BorgBackup 备份仓库以浏览文件。
  • mergerfs: 合并多个目录或文件系统为一个虚拟视图。
  • s3fs-fuse: 将 Amazon S3 存储桶挂载为文件系统。
  • NTFS-3G: (非 Windows 系统) 提供 NTFS 文件系统读写支持。

备份策略 (Backups)

任何没有备份的数据，都可能在任何时刻永远消失。

可靠地备份数据比简单地复制数据要复杂得多。

常见备份误区

  • 误区1: 同一磁盘上的副本是备份。 错误！磁盘是单点故障。
  • 误区2: 家中的外部硬盘是可靠备份。 不够可靠！火灾、盗窃等可能同时毁掉原数据和备份。
  • 误区3: 同步方案 (如 Dropbox) 等同于备份。 错误！文件删除或损坏会被同步，可能覆盖历史版本。
  • 误区4: RAID 是备份。 错误！RAID 提高可用性/性能，但不能防止删除、损坏或勒索软件。

良好备份方案的特性

  • 版本控制 (Versioning): 能够恢复到历史上的任意时间点。
  • 重复数据删除 (Deduplication): 仅存储变化的部分或共享的数据块，节省空间。
  • 加密 (Encryption): 保护备份数据的机密性，尤其是在异地或云端存储时（最好是客户端加密）。
  • 安全性 (Security): 控制谁能访问甚至删除你的备份。
  • 可恢复性验证 (Verifiability): 定期测试恢复流程，确保备份确实可用！
  • 自动化 (Automation): 自动执行备份，减少人为错误。

3-2-1 备份规则

一个经典的有效策略：

至少 3 份数据副本

存储在 2 种不同的介质上

至少有 1 份在异地

超越本地文件：云端数据备份

考虑备份存储在云服务上的重要数据（邮件、照片、文档等），以防账户丢失。可使用服务提供商的导出工具 (takeout.google.com) 或第三方工具 (rclone)。

推荐的备份工具

  • rsync: 文件同步基础工具。
  • Restic, BorgBackup, Kopia: 现代命令行工具，支持加密、去重、快照。
  • Time Machine (macOS): 用户友好，主要用于本地备份。
  • File History / Backup and Restore (Windows): 内置功能。
  • 云存储 (配合 Restic/Borg): Backblaze B2, S3 Glacier, Wasabi。
  • 云备份服务: Backblaze Personal, Carbonite。

API (应用程序编程接口)

许多在线服务都提供 API (Application Programming Interfaces)，允许你通过编程方式访问它们的数据和功能，而不是必须通过图形用户界面。这使得自动化、集成和构建基于这些服务的新应用成为可能。

API 的常见形式与特征

结构化 URL

通常基于 HTTP(S)，使用特定的 URL 路径和查询参数来指定要访问的资源或执行的操作 (如 api.service.com/users?id=123)。

响应格式

最常见的是 JSON (易于解析)。XML 也曾流行。

HTTP 方法

常用于指示操作类型 (RESTful 约定): GET (获取), POST (创建), PUT (替换), PATCH (部分更新), DELETE (删除)。

身份验证

通常需要验证调用者身份:

  • API 密钥 (API Keys)
  • OAuth (用于授权第三方应用)
  • JWT (JSON Web Tokens)

务必像保护密码一样保护好你的 API 密钥和令牌！

速率限制

API 通常会限制单位时间内的请求次数，防止滥用。

API 设计风格

  • REST (Representational State Transfer): 基于 HTTP 的主流架构风格，面向资源。
  • GraphQL: 查询语言，允许客户端精确请求所需数据，避免冗余。
  • gRPC: Google 开发的高性能 RPC 框架。

与 API 交互的工具

  • curl: 强大的命令行 HTTP 客户端。
  • Postman, Insomnia, HTTPie: 流行的 API 开发和测试工具。
  • jq: 命令行 JSON 处理工具。
  • IFTTT, Zapier: API 自动化和集成服务。

常用命令行标志/模式

虽然每个命令行工具都有其独特的选项，但在不同的工具之间也存在一些常见的标志和使用模式，了解它们可以帮助你更快地学习和使用新工具。

  • --help 或 -h (有时是 -?): 显示简要的帮助信息和用法。
  • --dry-run 或 -n: 试运行模式。打印将要执行的操作，但不实际执。用于预览潜在的破坏性操作。
  • --interactive 或 -i: 交互模式。在执行每个破坏性操作前提示用户确认。
  • --version 或 -V (有时是小写 -v): 显示程序的版本号。
  • --verbose 或 -v: 启用详细输出模式。可以多次使用（如 -vv, -vvv）来增加详细程度，便于调试。
  • --quiet 或 -q: 静默模式。减少输出信息，通常只在出错时打印。
  • 文件名位置的 -: 通常表示从标准输入 (stdin) 读取或写入到标准输出 (stdout)。
  • -r 或 -R: 递归操作。常用于 cp, rm 等命令，使其作用于目录及其内容。
  • -- (双破折号): 特殊参数，告知程序停止解析后面的参数为选项或标志。之后的所有内容都视为普通参数（即使以 - 开头）。例如，删除名为 -file.txt 的文件：rm -- -file.txt。
  • --output <format> 或 -o <file>: 指定输出格式 (如 json, yaml) 或将输出写入指定文件。

窗口管理器 (Window Managers)

你日常使用的桌面环境（如 Windows, macOS, Ubuntu 默认的 GNOME）通常包含一个浮动式 (floating) 窗口管理器。窗口可以自由移动、调整大小并相互重叠。

然而，尤其是在 Linux 世界中，存在多种窗口管理器类型。其中一种流行的替代方案是平铺式 (tiling) 窗口管理器。

平铺式窗口管理器

  • 窗口从不重叠，而是像瓷砖一样自动排列填满屏幕。
  • 打开新窗口时，现有窗口会自动调整大小以腾出空间。
  • 通常高度键盘驱动，允许你使用快捷键在窗口间导航、调整布局、移动窗口等，极大减少对鼠标的依赖。
  • 优点: 高效利用屏幕空间，高度可定制，适合键盘流用户和开发者。

流行的平铺式窗口管理器/工具

Linux
  • i3wm / Sway (Wayland)
  • AwesomeWM (Lua 配置)
  • dwm (极简, C 配置)
  • xmonad (Haskell 配置)
  • Hyprland (Wayland, 动画)

  • Yabai (需要部分禁用 SIP)

VPN (虚拟专用网络)

虚拟专用网络 (VPN) 在近年来变得非常流行，但理解它的实际作用和局限性很重要。

VPN 的作用 (理想情况下)

  • 更改公网 IP 地址和地理位置： 你的网络流量看起来像是从 VPN 服务器所在地发出的。
  • 加密本地网络流量： 在你和 VPN 服务器之间的连接是加密的，使得同一本地网络（如公共 Wi-Fi）的其他人无法轻易窥探你的流量内容。

注意事项与权衡

  • 信任转移： 你将信任从本地 ISP 转移到了 VPN 提供商。VPN 提供商可以看到你的 ISP 能看到的所有信息（例如你访问了哪些网站）。
  • HTTPS 的普及： 大部分敏感网络流量（银行、邮件等）已经通过 HTTPS 加密。VPN 在这种情况下对内容保密的额外作用有限（但仍能隐藏你访问的目标站点，对本地网络而言）。
  • 配置与安全风险： VPN 提供商可能配置错误，或使用过时/不安全的协议。
  • 隐私风险： 一些（尤其是免费）VPN 提供商可能会记录并出售用户数据。选择信誉不佳的 VPN 可能比不用更糟。
  • 性能影响： VPN 通常会增加延迟并可能降低带宽。

何时使用 VPN 可能有益

  • 连接不安全的公共 Wi-Fi 网络时。
  • 需要访问有地理位置限制的服务时（注意合规性）。
  • 需要远程访问公司或学校内部网络资源时。
  • 希望对本地 ISP 隐藏浏览目标时（信任 VPN 提供商）。

选择与自建建议

  • 选择信誉良好、有清晰且经审计的无日志政策的 VPN 提供商。
  • 考虑自建 VPN 以获得最大控制权。
  • WireGuard 是目前推荐的现代、快速、安全的 VPN 协议。可以使用 Algo VPN, Outline VPN 等项目简化自建过程。

Markdown

在你的编程生涯中，除了代码，你还会编写大量的文本文档：README、文档、笔记、博客文章、评论等。对于这些场景，使用 Word 或 LaTeX 这样的重型工具往往过于复杂。Markdown 是一种轻量级标记语言，它允许你使用易于阅读、易于编写的纯文本格式来添加格式化元素。

Markdown 的核心思想是尽可能地模仿纯文本电子邮件中常见的格式化约定。

基本语法回顾

  • *斜体* 或 _斜体_
  • **粗体** 或 __粗体__
  • # 一级标题, ## 二级标题 ...
  • - 无序列表项 (或用 *, +)
  • 1. 有序列表项
  • `行内代码`
  • 代码块 (缩进或用 ``` 包裹)
  • [链接文本](URL)
  • ![图片描述](URL)
  • > 引用块
  • --- 或 *** (水平线)

许多平台还支持扩展语法，如 GitHub Flavored Markdown (GFM)，它增加了表格、任务列表 (- [x])、删除线 (~~文本~~) 等。

应用与工具

Markdown 应用极其广泛：项目 README、文档生成、笔记应用、博客平台、论坛、聊天软件等。本课程的所有讲义也是用 Markdown 编写的。

  • 编辑器: VS Code (内置强大支持), Typora, Obsidian, Joplin, Simplenote。
  • 转换工具: Pandoc 是一个强大的文档格式转换工具，可以在 Markdown 和 HTML, PDF, LaTeX, Word 等多种格式间转换。

启动与 Live USB

计算机启动时，在加载操作系统之前，会先运行一段固件程序，称为 BIOS (基本输入输出系统) 或更现代的 UEFI (统一可扩展固件接口)。这个固件负责初始化硬件并加载操作系统的引导程序。

  • 你通常可以在开机时按特定按键 (如 F2, F9, F12, Del，具体按键因主板而异) 进入 BIOS/UEFI 设置界面，可以配置硬件选项、启动顺序等。
  • 启动菜单允许你选择从哪个设备启动，例如硬盘、U 盘或光驱。

Live USB (可启动 U 盘)

Live USB 是指包含一个完整、可运行的操作系统的 U 盘。你可以从 Live USB 启动计算机，而无需修改硬盘上已安装的系统。

创建与使用

  1. 下载你想要使用的操作系统的 .iso 镜像文件 (例如 Ubuntu, Fedora, Arch Linux 等发行版)。
  2. 使用专门的工具将 .iso 文件"烧录"到 U 盘上。这不仅仅是复制文件。
    • 推荐工具: Ventoy (可启动多个 ISO), balenaEtcher, Rufus (Windows)。
    • 命令行: dd (Linux/macOS, 需谨慎)。
  3. 重启计算机，进入启动菜单，选择从 U 盘启动。

用途

  • 系统救援与恢复: 当主系统无法启动时，用于访问文件、修复引导或重装系统。
  • 试用操作系统: 无需安装即可体验 Linux 发行版或其他系统。
  • 硬件诊断: 运行内存测试 (memtest86+) 或其他硬件检测工具。
  • 安全与隐私: 运行专注于安全或隐私的发行版，如 Kali Linux (渗透测试) 或 Tails (匿名)。

虚拟化与云计算

虚拟机 (VMs)、容器 (Containers) 以及云计算是现代软件开发和部署中不可或缺的技术，它们提供了强大的环境隔离、可复现性和资源弹性。

虚拟机 (VMs)

  • 模拟完整的硬件系统，运行独立的操作系统内核。
  • Hypervisors: VirtualBox, VMware Workstation/Fusion, Parallels, Hyper-V, KVM。
  • 优点: 强隔离性，可运行不同操作系统。
  • 缺点: 资源开销较大。

Vagrant

  • 用于构建和管理虚拟开发环境的工具。
  • 通过 Vagrantfile 定义机器配置。
  • 使用 vagrant up 快速创建和配置 VM。
  • 优点: 标准化、可复现的开发环境。

容器 (Containers - Docker)

  • 在操作系统层面虚拟化，共享宿主机内核，但有隔离的文件系统、进程、网络。
  • 优点: 轻量、快速启动、资源占用少。
  • Docker 核心: 镜像 (Images), 容器 (Containers), Dockerfile, 仓库 (Registry), docker-compose。
  • 应用: 微服务、CI/CD、环境一致性。
  • 编排: Kubernetes (K8s) 是主流的容器编排平台。

云计算 (Cloud Computing)

  • 通过互联网按需租用计算资源。
  • 主要提供商: AWS, Azure, GCP, DigitalOcean 等。
  • 优点: 弹性伸缩、按需付费、访问全球资源。
  • 服务模型: IaaS, PaaS, SaaS, FaaS (Serverless)。

OpenStack

  • 开源的云计算平台软件。
  • 允许组织构建自己的私有云或公有云。

Notebook 编程

Notebook 编程环境提供了一种交互式的计算方式，特别适合数据探索、可视化、教学演示和快速原型设计。它允许你在一个文档中混合代码、文本、数学公式和可视化结果。

Project Jupyter

目前最流行的 Notebook 环境生态。

  • Jupyter Notebook: 经典的基于 Web 的界面。
  • JupyterLab: 下一代界面，提供更集成和灵活的开发环境。
  • Kernels: 支持多种编程语言（Python 是核心，但也支持 R, Julia, Scala 等）。
  • Google Colaboratory (Colab): 免费的云端 Jupyter 环境，提供 GPU/TPU 访问，特别适合机器学习。
  • VS Code 集成: VS Code 提供了优秀的内置 Jupyter Notebook 支持。

其他 Notebook 环境

  • Wolfram Mathematica: 强大的符号计算和可视化环境。
  • Apache Zeppelin: 主要面向大数据生态。
  • Observable: 专注于 JavaScript 数据可视化。

应用场景

  • 数据科学与分析
  • 机器学习模型开发与实验
  • 教学与演示
  • 快速原型验证
  • 科学计算

Hammerspoon (macOS 桌面自动化)

Hammerspoon 是一个强大的 macOS 桌面自动化框架。它允许你使用 Lua 脚本与操作系统底层 API 进行交互，从而实现对键盘、鼠标、窗口、文件系统、音频、网络等几乎所有方面的控制和自动化。

Hammerspoon 能做什么？

  • 绑定全局快捷键，快速将窗口移动到屏幕特定位置或不同显示器。
  • 创建自定义菜单栏图标，点击后执行特定脚本（如按预设布局排列窗口）。
  • 根据 Wi-Fi 网络自动切换音频输出或执行其他操作。
  • 当特定 USB 设备连接或断开时触发动作。
  • (几乎)任何你能想到的与系统交互相关的自动化任务。

学习与资源

  • 官方入门指南
  • 示例配置
  • 搜索 GitHub 查看其他用户的 Hammerspoon 配置 (dotfiles)。
  • 讲师 Anish 的 Hammerspoon 配置。

其他平台替代方案

  • Linux: AutoKey, xdotool, ydotool (Wayland), Python 脚本 (pynput, pyautogui) 等。
  • Windows: AutoHotkey (非常强大)。

GitHub 与开源贡献

GitHub 是目前最流行的代码托管和开源协作平台之一。本课程中提到的许多工具都托管在 GitHub 上。参与开源项目是学习、提升技能、建立联系并回馈社区的绝佳方式。

参与开源的两种主要方式

创建 Issue (问题)

  • 报告 Bug (提供清晰复现步骤、环境信息)。
  • 请求新功能 (Feature Request)。
  • 提问或参与讨论。
  • 通常不需要编写代码，参与门槛较低。
  • 关注标签如 good first issue, help wanted。

贡献代码 (Pull Request - PR)

通常涉及代码编写和修改。基本流程 (Fork & PR):

  1. Fork 目标仓库到你的账户。
  2. Clone 你的 Fork 到本地。
  3. 创建新的分支进行修改。
  4. Commit & Push 修改到你的 Fork。
  5. 在 GitHub 上创建 Pull Request，将你的分支提交到原始仓库。

之后会进行代码审查 (Code Review)。

贡献前注意

  • 阅读项目的 CONTRIBUTING.md (贡献指南)。
  • 遵守项目的代码风格和规范。
  • 确保你的提交包含测试（如果适用）。

其他代码托管平台

  • GitLab
  • Bitbucket
  • Gitee (码云)
